# 사용자 관점 평가

## 사용자가 좋게 느낄 점

1. **장면을 이해한 답변처럼 보인다.** “라이트룸/인플루언서” 같은 내부 용어가 아니라, 창가 인물·비 오는 네온·콘서트·눈 사진 같은 장면에 맞춰 답이 나온다.
2. **트렌드/일반/개인화가 분리되어 있어 납득하기 쉽다.** 사용자는 “기본 안전한 답”과 “요즘 느낌”과 “내 취향에 맞춘 변형”을 구분해서 받을 수 있다.
3. **촬영과 보정이 같이 나온다.** 다시 찍을 때의 렌즈/구도/빛, 이미 찍은 사진의 마스크/노출/색보정이 같이 제공된다.
4. **수치가 있어 실행 가능하다.** EV, Highlights, Saturation, Texture, Grain 같은 시작값이 있어 막연한 조언보다 실용적이다.
5. **근거 파일이 붙는다.** 추천이 어디에서 왔는지 추적 가능하다.

## 사용자가 답답하게 느낄 점

1. **답변이 아직 ‘그래프 노드 문장’ 같다.** 일부 technique/parameter가 원문 bullet 그대로라 문장 흐름이 딱딱하다.
2. **이미지에서 실제 문제가 감지된 것은 아니다.** 현재는 텍스트 질문을 scenario에 매칭한 것이고, 실제 이미지 분석 기반으로 `face underexposed`, `horizon tilted` 등을 자동 감지한 것은 아니다.
3. **개인화는 아직 가정 수준이다.** `prefers_warm`, `prefers_cinematic` 같은 노드는 있지만 사용자 히스토리/선호 점수는 아직 없다.
4. **질문이 scenario와 정확히 매칭되지 않으면 답이 부정확할 수 있다.** 예를 들어 플랫레이 음식 질문이 marketplace street food로 매칭되는 식의 근사 매칭이 생긴다.
5. **보정값이 사진별로 과할 수 있다.** 시작값은 유용하지만, 실제 히스토그램/피부톤/조명 분석 없이 고정값을 주면 과보정 위험이 있다.

## 사용자가 받을 인상

- “오, 그냥 출처 검색이 아니라 사진 코치처럼 답하려는 구조는 맞다.”
- “근데 실제 이미지 기반 분석이 붙어야 진짜 전문가처럼 느껴질 것 같다.”
- “개인화 추천이라고 하지만 아직은 내 취향을 실제로 안다는 느낌은 약하다.”
- “답변을 앱에서 바로 따라 할 수 있게 Before/After 단계나 슬라이더 카드 형태로 보여주면 훨씬 좋겠다.”

## 다음 개선 우선순위

1. **이미지 분석 facet 자동 추출**: subject, environment, light, issue, crop, face brightness, sky clipping, dominant colors.
2. **질문 → scenario 매칭 개선**: 한국어 자연어 임베딩/키워드 사전/Neo4j fulltext index.
3. **추천 후처리 템플릿**: 그래프 노드를 사용자 문장으로 자연스럽게 변환.
4. **개인화 데이터 추가**: 사용자가 좋아요/저장/선택/무시한 추천을 Preference와 SatisfactionSignal에 연결.
5. **보정값 안전장치**: 히스토그램/피부톤/화이트밸런스 감지 후 수치 범위를 좁히기.
